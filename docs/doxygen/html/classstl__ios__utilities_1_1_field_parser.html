◆ parse_fields()

std::istream& stl_ios_utilities::FieldParser::parse_fields ( std::istream *  is,
std::vector< std::string > *  fields,
int  field_number = 1 
)

Reads a characters from is splitting the contents into fields.

Reads characters from is until either one of the terminators is read, or the stream evaluates to false. Whenever a field delimiter is read std::string object consisting of accumulated characters is parsed using the function, if any, mapped to by the respective column number in field_parsers_. Fields read replace the contents of fields.

Enforces requested number of fields (field_number) by throwing an exception of type stl_ios_utilities::MissingFields if fewer fields than than requested are read when the end of file of a terminator is read, and enforce_field_number_ is set to true (default). In this case, fields remains unchanged and is remains in a valid state. If too few fields are read and enforce_field_number_ is set to false, the fields will be stored in fields unless ignore_underfull_data_ is set to true (default). In the latter case, fields is not modified.

Characters listed in data member masked_ are skipped while reading from is.

Field parsers mapped to by field numbers in field_parsers_ may throw exceptions depending on the function. Internally, this method repeatedly uses std::istream::get to read the row character-by-character, which may throw exceptions (see STL docs).

Concurrent access to is may cause data races as documented in STL docs for std::istream::get.

Parameters
isPointer to the input stream containing delimited data.
fieldsstd::vector<std::string> object in which the fields are stored, if any.
field_numberThe number of fields requested to be read into fields (default: 1).
Returns
Returns a reference to the input stream is after extraction of the fields. If an exception of type stl_ios_utilities::BaseException occurred, or only fewer fields than requested are read, then is is left at the point where parse_fields stopped reading characters.
